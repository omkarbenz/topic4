SIT209: IoT Application Development

Activity 1

An IoT (Internet of Things) application refers to a software program or system that enables the connectivity, monitoring, and control of devices, objects, or "things" in an IoT ecosystem.

    Exmaples of IoT applications include:
  • Smart homes: systems that allow for remote control of lights, temperature, and security systems
  • Wearables: fitness trackers, smartwatches, and other wearable devices that collect and transmit data
  • Industrial IoT (IIoT): applications that monitor and control industrial equipment and processes
  • Healthcare IoT: devices that collect and transmit patient data for remote monitoring and analysis
  • Connected vehicles: cars that transmit data to support safety, navigation, and maintenance functions

    The main features needed in an IoT application include:
  • Connectivity: the ability to connect to and communicate with IoT devices and networks
  • Data collection and analysis: the ability to collect and process data from IoT devices
  • Real-time monitoring and control: the ability to monitor and control IoT devices in real-time
  • Scalability: the ability to support a large number of IoT devices and networks
  • Security: the ability to protect against unauthorized access and data breaches in IoT systems.

Activity 2

"Smart" building refers to a building equipped with advanced technology and automation systems that allow it to optimize its energy consumption, enhance comfort and safety, and improve operational efficiency.

    Some smart building scenarios where this technology would be beneficial include:
  • Automated lighting and temperature control to conserve energy and improve comfort
  • Advanced security systems to monitor and protect the building
  • Intelligent elevators to optimize traffic flow and reduce wait times
  • Advanced HVAC systems to monitor and control indoor air quality

    The parts needed for a smart building to work include:
  • Building Management System (BMS) to control and monitor all systems
  • Internet of Things (IoT) devices and sensors to gather data and control systems
  • Network infrastructure to connect all systems and devices
  • User interfaces such as touchscreens and mobile apps for easy control and monitoring.

Activity 3

    For a smart building IoT application, a control interface should have the following high-level features:
  • User-friendly interface: The interface should be easy to navigate and use for building administrators, facility managers, and end-users.
  • Real-time monitoring: The interface should provide real-time monitoring of the building systems such as HVAC, lighting, and security.
  • Remote control: The interface should allow users to remotely control building systems, such as adjusting the temperature or turning on lights.
  • Energy management: The interface should provide energy management features, such as setting energy efficiency goals, monitoring energy usage, and controlling energy-consuming systems.
  • Alerts and notifications: The interface should provide alerts and notifications for issues such as system failures, energy usage thresholds, and security breaches.
  • Data analytics: The interface should provide data analytics, such as occupancy patterns and energy usage patterns, to help building administrators make informed decisions about building operations.

    To make this work, the following parts are needed:
  • IoT devices: Smart sensors and actuators that are installed throughout the building to collect and transmit data.
  • Network infrastructure: A reliable and secure network infrastructure that connects the IoT devices to the control interface.
  • Cloud platform: A cloud-based platform that stores and processes the data collected by the IoT devices and provides the control interface.
  • Data management system: A system for managing and organizing the data collected by the IoT devices.
  • User authentication and authorization: A system for authenticating users and authorizing access to the control interface and building systems.
